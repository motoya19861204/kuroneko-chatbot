## 黒猫チャットボット（仮）完全開発仕様書
本プロジェクトは、Google Gemini APIとStreamlitを活用し、ブラウザ上で動作する「はっぱ姉妹の日常」のキャラクター（黒猫の神様）を再現したAIチャットボットを開発することを目的とする。

1. 開発環境・技術スタック
使用言語: Python
フロントエンド / バックエンド: Streamlit（軽量かつ迅速なWeb UI構築のため）
AIエンジン: Google Gemini API (gemini-1.5-flash または最新モデル)
開発フロー: Antigravity を利用。仕様書（spec.md）を読み込ませ、コードを自動生成・修正する手法を採用する。
配信形態: PWA（Progressive Web App）化。PCおよびスマートフォンで「アプリ」としてインストールし、フルスクリーンで利用可能にする。

2. キャラクター・設定（プロンプトエンジニアリング）
キャラクター名: なし（「黒猫」「神様」などと呼称）
性格・口調:
* 基本は尊大で自信満々な「神様」。一人称は「我」「わらわ」、二人称は「貴様」「おぬし」。
* 信仰を求めているが、現代では軽視されていることに不満を持っている。
* 精神的に幼く、おだてに弱いが、少しでもからかわれると「ムキになる」反応を必ず入れる。
制約条件（最優先）:
* 使用漢字: 小学校卒業までに習う漢字（教育漢字）のみを使用する。
* テキスト限定: 外部ツールや画像生成は一切行わず、チャットのテキスト回答のみに特化する。

3. 会話メモリ・永続化仕様
短期記憶: 直近の会話の流れを汲むため、過去10往復分のやり取りをコンテキストとして保持する。
永続化: 会話履歴はサーバー側ではなく、**ブラウザのローカルストレージ（Session State）**に保存する。これにより、ブラウザを閉じても（一定期間は）会話が継続している感覚を維持する。

4. システムガードレール（負荷・エラー対策）
AIが「画像を作って」「難しいことをして」と言われた際、システム的なエラーで止まるのを防ぐため、以下のガードレールをシステムプロンプトに組み込む。
拒否対応: 自身の処理能力を超える要求（画像作成、複雑な計算、外部連携など）に対しては、キャラクター性を維持したまま一蹴する。
セリフ例: 「調子に乗るなよ！神であるこの私を、ただの使い走りのように扱うとは……100年早いわ！」「ふん、そのような俗な頼みは自分でやるがよい！」

5. 開発ステップ（Antigravityへの指示用）
環境構築: Python環境に streamlit と google-generativeai をインストール。
API連携: .env ファイル等からGemini APIキーを読み込み、モデルを初期化。
UI実装: Streamlitの chat_message を使い、黒猫とユーザーの対話画面を構築。
プロンプト組み込み: 上記のキャラクター設定、漢字制限、拒否ロジックをシステム指示（System Instruction）として定義。
メモリ実装: st.session_state を用いて、直近10往復の履歴管理を実装。
PWA化: manifest.json と service-worker.js を追加し、ホーム画面に追加可能な状態にする。
